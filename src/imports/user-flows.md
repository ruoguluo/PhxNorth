Design two critical user flows:

1) Role Selection Page
2) Mentee Question Entry Engine

--------------------------------------------------
FLOW 1 — ROLE SELECTION
--------------------------------------------------

Scenario A: New user with no activated roles.

Create a full-page role selection screen.

Title:
Choose Your Role

Subtitle:
Activate one or more growth pathways.

Display 3 role cards:

• Mentee
• Mentor
• Business Consultant

Each card includes:
• Short value proposition
• Key benefits
• Activate button

Allow multi-selection.

After activation:
Route to selected role dashboard.

--------------------------------------------------
Scenario B: Existing user with partial roles.

Before entering dashboard, show modal:

"Unlock Additional Roles"

Display unactivated roles with:
• Short benefit summary
• Activate button
• Skip option

Do not force activation.

--------------------------------------------------
FLOW 2 — MENTEE QUESTION ENTRY
--------------------------------------------------

On Mentee dashboard, create a prominent entry section:

Title:
What would you like to solve today?

Two large buttons:

• Structured Question
• Quick Question

--------------------------------------------------
STRUCTURED QUESTION FLOW
--------------------------------------------------

Step 1 — Define Objective

Collect:
• Domain
• Background context
• Desired outcome
• Time horizon
• Success criteria

--------------------------------------------------
Step 2 — AI Decomposition

Based on:
• Industry
• Question type
• User background
• Growth stage

Generate structured agenda:

• Sub-question 1
• Sub-question 2
• Sub-question 3

Each includes:
• Purpose
• Depth level
• Estimated discussion time

Allow editing and reordering.

--------------------------------------------------
Step 3 — Confirmation

Display:
• Goal clarity level
• Structural depth score

Button:
Start Structured Mentorship

--------------------------------------------------
QUICK QUESTION FLOW
--------------------------------------------------

Allow direct question input.

System auto-detects:
• Domain
• Intent
• Complexity

Offer suggestion:
"Structure this question for better outcome?"

--------------------------------------------------
DESIGN PRINCIPLES
--------------------------------------------------

• Clean SaaS layout
• Step-based wizard
• Emphasize goal-driven problem solving
• No casual Q&A tone
• Focus on structured decision infrastructure
