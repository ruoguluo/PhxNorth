Design an upgraded Mentor Operations Center for an AI-powered mentorship platform.

This is not a passive dashboard.
It is an action-driven control hub.

--------------------------------------------------
GLOBAL PAGE CONCEPT
--------------------------------------------------

The page must encourage:

• Role activation (headhunter / consultant / workshop speaker)
• Real-time decision actions
• Structured mentorship workflow control
• System-generated next-step reminders

--------------------------------------------------
SECTION 1 — ROLE ACTIVATION PANEL (NEW SECTION ABOVE METRICS)
--------------------------------------------------

Add a new horizontal panel below the main title:

Title:
Unlock More Impact & Income

Display 3 role cards if not activated:

1. Talent Referral (Headhunter)
   - Short description
   - Estimated earning potential
   - Badge: “Not Activated”
   - Button: Activate Role

2. Enterprise Consulting
   - Short description
   - Badge: Opportunities Available
   - Button: View Projects

3. Workshop Speaker
   - Short description
   - Badge: 2 Invitations Pending
   - Button: Review Invitations

If role already activated:
Show green “Active” label instead of button.

This section should feel like growth opportunity prompts.

--------------------------------------------------
SECTION 2 — METRICS CARDS (KEEP BUT SLIGHTLY REFINED)
--------------------------------------------------

Keep:

• Sessions this week
• Active mentees
• Average rating
• Monthly income

Add small action link inside each card:
“View Details”

--------------------------------------------------
SECTION 3 — REAL-TIME QUEUE CENTER (REPLACES INSTANT MENTORSHIP QUEUE)
--------------------------------------------------

Rename section title:

Mentorship Activity Queue

Subtitle:
Live requests & structured session actions

This section must be tab-based.

Tabs:

• Instant Mentorship Queue
• Scheduled Mentorship Requests
• Action Required

--------------------------------------------------
TAB 1 — INSTANT MENTORSHIP QUEUE

Display real-time waiting users:

Each card includes:

• Avatar
• Name
• Big question preview
• Estimated duration
• 5D quick match score
• Wait time indicator
• Accept button

--------------------------------------------------
TAB 2 — SCHEDULED MENTORSHIP REQUESTS

List newly submitted structured requests.

Each card includes:

• Avatar
• Name
• Big question preview
• AI estimated total time
• Status:
  - Awaiting Review
  - Awaiting Mentee Confirmation

• Button: Review & Structure

--------------------------------------------------
TAB 3 — ACTION REQUIRED (VERY IMPORTANT)

This tab shows system-triggered reminders.

Examples:

• Mentee replied to Question 2
• Session starts in 24 hours
• Project milestone needs review
• Mentee confirmed structured agenda
• Waiting for mentor feedback

Each reminder card includes:

• Context label (Session / Project / Instant)
• Mentee name
• Action required description
• Primary action button (Reply / Review / Confirm)

This tab must feel urgent and prioritized.

--------------------------------------------------
SECTION 4 — SYSTEM PROJECT & WORKFLOW REMINDERS (NEW BELOW QUEUE)
--------------------------------------------------

Add a section titled:

Next Structured Actions

Display structured progress blocks:

Example cards:

• Big Question 60% complete
• Sub-question 3 pending mentor input
• Enterprise project deliverable due in 2 days
• Workshop outline awaiting approval

Each card includes:

• Progress bar
• Status label
• Action button

This reinforces structured workflow control.

--------------------------------------------------
DESIGN STYLE

• Clean SaaS layout
• Soft green primary accent
• Orange for pending actions
• Red for urgent
• Blue for system intelligence

Must visually communicate:

This is a command center.
The mentor is managing structured decision processes,
not just attending sessions.
